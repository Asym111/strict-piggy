# 🐷 Строгая Копилка (StrictPiggy)

Приложение для накопления денег с жёсткой дисциплиной: пополняй копилку **каждый день** — или теряй **10% накоплений** за каждый пропущенный день. Реализовано по дизайн-ТЗ `StrictPiggy_Design_TZ.docx` (версия правил 1.0 от 01.07.2026).

## 🚀 Запуск

Это статическое веб-приложение без зависимостей — просто откройте `index.html` в браузере, либо запустите локальный сервер:

```bash
python -m http.server 8080
# → http://localhost:8080
```

Все данные хранятся локально в `localStorage` браузера.

## ⚡ Функции

- **Онбординг** — 3 слайда с объяснением концепции и правил.
- **Вход** — имитация Google Sign-In (демо-режим, без реального OAuth).
- **Экран согласия на штрафы** — красный предупреждающий текст, обязательный чекбокс (кнопка неактивна без галочки), повторное подтверждение кодом (имитация email-кода). Согласие логируется: дата, IP, версия правил (видно в настройках).
- **Создание цели** — название, сумма, срок; автоматический расчёт ежедневного взноса.
- **Дашборд** — баланс, прогресс-бар, процент выполнения, серия дней 🔥, счётчик оставшихся дней, сумма штрафов, обратный отсчёт до полуночи.
- **Ежедневное пополнение** — одно пополнение в день; кнопка блокируется после взноса.
- **Штрафы 10%** — за каждый полностью пропущенный день автоматически списывается 10% текущего баланса «в пользу владельца приложения»; модальное окно штрафа, сброс серии. Необратимо.
- **История операций** — все пополнения, штрафы и выводы.
- **История штрафов** — отдельный экран с суммой потерь.
- **Вывод средств** — доступен только после достижения цели.
- **Отзыв согласия** — удаление аккаунта с выводом остатка и потерей 50% (подтверждение словом «УДАЛИТЬ»).
- **Стили уведомлений** — «жёсткий» 😤 и «мотивирующий» 🌤, переключаются при создании цели и в настройках.
- **Демо-режим** — кнопка «Симулировать следующий день» в настройках для проверки логики штрафов без ожидания.

## 🧱 Структура

```
index.html      — все экраны приложения (SPA)
css/style.css   — тёмная тема, стили экранов и модалов
js/app.js       — состояние, логика штрафов, рендер, навигация
```

## ⚠️ Статус прототипа

Это **клиентский прототип** (демо): деньги виртуальные, авторизация и email-код имитируются, платёжные интеграции и backend отсутствуют. По ТЗ продакшен-версия предполагает Flutter + Firebase (Auth, Firestore, Cloud Functions для начисления штрафов, push-уведомления) и платёжного провайдера. Для реального продукта с удержанием средств пользователей потребуется юридическая проверка (лицензии на приём платежей, законность безакцептных списаний в вашей юрисдикции).
